# Contributing to Enigma Cipher

Thank you for your interest in contributing to the Enigma Cipher project! This document provides guidelines for contributing to this historical cryptography implementation.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or fix
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

## Code Standards

- Write clear, readable code with appropriate comments
- Follow the existing code style and structure
- Ensure historical accuracy when implementing cipher features
- Include documentation for new features

## Testing

- Test all cryptographic operations thoroughly
- Verify that encryption/decryption cycles work correctly
- Test edge cases and invalid inputs
- Ensure backward compatibility with existing functionality

## Commit Messages

- Use clear, descriptive commit messages
- Reference related issues when applicable
- Keep commits focused on single logical changes

## Pull Requests

- Provide a clear description of the changes
- Link to any related issues
- Ensure all tests pass
- Be responsive to feedback and requested changes

## Reporting Issues

- Use the issue tracker for bugs and feature requests
- Provide detailed information about the problem
- Include steps to reproduce bugs
- Suggest solutions if possible

## Questions?

Feel free to open an issue for questions about contributing.
